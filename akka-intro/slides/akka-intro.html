So, what is the Actor model?

Actor's definition

Fundamental unit of computation that embodies:

  • Processing
  • Storage
  • Communication

Three axioms. When an Actor receives a message it can:

  • Create new Actors
  • Send messages to Actors it knows
  • Designate how it should handle the next message it receives

Actor structure

  • No synchronization is required
  • Extremely lightweight (300 bytes each aprox)
  • Asynchronous

    Define an actor

    public class MetricsProcessorActor extends UntypedActor {

	public void onReceive(Object message) {
        if (message instanceof JVMMetric) {
        	JVMMetric jvmMetric = (JVMMetric)message;

        	_dataStore.save(jvmMetric);
        }
	}
    private DataStore _dataStore = new LogDataStore();
}

	public void onReceive(Object message) {

            AuthMessage authMessage = (AuthMessage)message;

            	getSender().tell("ok");

    Futures

    Reason about performing many operations in parallel. In an efficient and non-blocking way

    Abstraction which represents a value which may become available at some point

    Inmutable! Holder cannot write or failed the future

    An ExecutionContext is required in order to run a future

    Creating a future

    import scala.concurrent.duration.Duration;
import akka.japi.Function;
import java.util.concurrent.Callable;
import static akka.dispatch.Futures.future;
import static java.util.concurrent.TimeUnit.SECONDS;
		
    Future f = future(new Callable() {
  public List call() {
    return twitterClient.listTweets();
  }
  private final TwitterClient twitterClient = new TwitterClient();
}, system.dispatcher());
 
f.onSuccess(new PrintResult(), system.dispatcher());

    Callbacks

    Interested in the result of the computation; not only side effects

    Able to register a callback. Completely non blocking!

    Register callbacks

    val f: Future[List[String]] = future {
  session.getRecentPosts
}

// handle both cases: Success and Failure
f onComplete {
  case Success(posts) => for (post <- posts) println(post)
  case Failure(t) => println("An error has occured: " + t.getMessage)
}		
// only handle Success
f onSuccess {
  case posts => for (post <- posts) println(post)
}
// only handle Failure
f onFailure {
  case t => println("An error has occured: " + t.getMessage)
}
